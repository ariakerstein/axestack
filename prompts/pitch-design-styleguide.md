# Navis Pitch Deck - Design & Style Guide

## Typography

### Fonts
- **Headings**: Inter Bold (700 weight)
- **Body Text**: Inter Regular (400 weight)

### Sizes
- **Headline**: ~36–44 pt (adjust for tool scaling)
- **Body**: ~20–24 pt (adjust for tool scaling)

## Color Palette

### Primary Colors
- **Background**: `#FFFFFF` (white)
- **Text**: `#0A0A0A` (near black)
- **Accent/Brand**: `#0057FF` (Navis Blue)
- **Highlight/Growth**: `#00C48C` (green)

### Usage Guidelines
- Use Navis Blue (`#0057FF`) for:
  - Call-to-action elements
  - Key highlights
  - Gradient backgrounds (with complementary blues)
  - Progress indicators
- Use green (`#00C48C`) for:
  - Growth metrics
  - Positive outcomes
  - Completion states

## Layout Principles

### Slide Structure
- **One idea per slide** - don't overcrowd
- **3–5 bullets maximum** per slide
- **Large visuals** - images, diagrams should be prominent
- White space is your friend

### Visual Hierarchy
1. Main headline (largest, boldest)
2. Supporting subheading (medium, lighter weight)
3. Body copy or bullets (smallest, regular weight)

## Visuals

### Preferred Visual Types
- **Custom icons/diagrams** rather than stock photos when possible
- High-resolution images only
- Consistent icon style across deck

### Charts & Data Visualization
- **Clean design** - remove unnecessary gridlines and decorations
- **Annotate the headline number** - make the key metric obvious
- Use Navis Blue for primary data series
- Use green for growth/positive trends

### Screenshots
- **High resolution only** - no pixelation
- **Limited text** - show, don't tell
- **Call out key parts** - use arrows, highlights, or borders
- Crop tightly to relevant area

## Tone & Voice

### Writing Style
- **Bold, simple sentences** - get to the point
- **Avoid jargon** - explain in plain language
- **Use "we"** not "I" - team, not individual
- **Maintain crisp focus** - one concept at a time

### Language Choices
- Use **"we will"** not "we may" or "we could" (no pivot language)
- Be **declarative and confident**
- **Active voice** over passive voice

## Narrative Framework

### Core One-Liner
> "Cancer is an expedition. Navis builds the map, coordinates the team, and learns from every journey to guide the next."

### Metaphor Guidelines
- Use metaphors that are **vivid but explain mechanics**
  - Examples: "map", "team", "trail", "expedition", "coordinate"
- Avoid metaphors that confuse or require explanation

### Content Focus
- **Focus on outcomes** (what happens) not just features (what you build)
- **Use data to support claims** (e.g., "80% of care coordination happens outside the clinic")
- **Show, don't just tell** - use visuals to demonstrate concepts

### Founder Story
- Include **only if uniquely compelling**
- Keep brief (1-2 slides maximum)
- Focus on **intrinsic motivation** and relevant expertise
- Currently omitted for brevity

## File Management

### Source Files
- **Working files**: `prompts/` directory
- **Production files**: `public/` directory
- **Assets**: `public/pitchAssets/` directory

### Naming Conventions
- Use descriptive, lowercase names
- Separate words with hyphens: `navis-tbd-angels-pitch.html`
- Version with dates if needed: `pitch-2025-01-15.html`

## Technical Specifications

### HTML Pitch Decks
- Use Tailwind CSS for styling
- Inter font from Google Fonts
- Responsive design (scales to different screen sizes)
- Print-optimized (each slide = one PDF page)
- Keyboard navigation enabled

### Image Assets
- **Format**: JPG for photos, PNG for graphics with transparency
- **Resolution**: Minimum 1920px width for full-slide images
- **Optimization**: Compress without visible quality loss
- **File size**: Keep under 200KB per image when possible
